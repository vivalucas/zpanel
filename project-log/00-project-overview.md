# 项目概述

## 项目名称

ZPanel

## 项目背景

ZPanel fork 自 [Sun-Panel](https://github.com/hslr-s/sun-panel)。原项目采用 MIT 协议，允许复制、修改、分发和再发布，但需要保留原版权声明和许可证文本。

用户长期使用 Sun-Panel 作为个人服务器 / NAS 导航面板。由于上游开源版本维护节奏未满足用户预期，且后续 PRO / 付费相关规划没有按用户预期持续公开更新，因此本项目以 MIT fork 为基础建立独立维护分支。ZPanel 按全新项目发布，不承诺兼容上游旧配置导入；当前目标是在保留上游许可证归属的前提下，完成独立品牌、构建链路、维护文档和功能演进基线。

2026-05-21，GitHub 仓库已从“GitHub fork 仓库”调整为重新创建的同名独立仓库，并以精简后的初始提交重新推送。因此当前 GitHub 上不再显示 fork 标识、Sync fork 或旧历史。该动作只改变仓库形态和公开历史，不改变 ZPanel 来源于 MIT 许可 Sun-Panel 开源版本的事实；README / LICENSE 中仍需保留上游归属说明。

## 用户 / 使用场景

- 个人服务器、NAS、Homelab 用户：集中管理常用服务入口。
- 浏览器首页用户：把导航面板作为日常访问入口。
- 自托管爱好者：通过 Docker 或二进制部署到本地服务器。
- 后续维护者 / AI 助手：基于 project-log 快速理解 fork 背景、当前状态和技术边界。

## 核心功能

1. 导航项目与分组管理。
2. 多账号隔离和公开访问模式。
3. 内外网访问地址切换。
4. 系统状态展示。
5. 自定义样式、背景、页脚、登录页、站点图标、JS 和 CSS。
6. 图标上传、公共图库、站点 favicon 获取、Iconify 图标支持。
7. 数据导入导出，包含导航图标与样式配置。
8. Docker / 二进制方式部署，默认端口 `6521`。
9. 管理员 Docker 容器管理。
10. `/api/healthz` 健康检查，用于 Docker / Compose / CI 基础验证。

## 核心概念

- **导航项目**：面板上的应用 / 服务入口，包含名称、地址、图标、描述、排序等信息。
- **项目分组**：导航项目的分类容器，用于组织不同服务。
- **公开模式**：允许未登录访问部分面板数据的模式，具体权限由后端拦截器控制。
- **内外网模式**：同一导航项目可配置不同访问地址，在局域网和公网环境间切换。
- **系统配置**：存储于数据库中的站点级配置，例如登录配置、免责声明、关于说明等。
- **站点设置**：站点标题、favicon、登录页文案、登录验证码、自定义 CSS / JS 等全局展示配置。
- **公共图库**：管理员和用户上传图片形成的公共可选资源视图；删除仍按当前用户权限处理。
- **Docker 应用管理**：管理员通过后端调用 Docker CLI 管理宿主机容器，依赖部署环境提供 Docker 权限。

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端 | Go 1.26.3、Gin、GORM | HTTP API、静态文件服务、数据库访问 |
| 前端 | Vue 3、Vite、TypeScript、Pinia、Vue Router | 单页应用 |
| UI | Naive UI、Tailwind CSS、Iconify | 组件和图标能力 |
| 数据库 | SQLite 默认，MySQL 可选 | GORM 自动迁移 |
| 缓存 | Memory 默认，Redis 可选 | 用户 token、限流等缓存 |
| 队列 | Memory 默认，Redis 可选 | 后端队列能力 |
| 部署 | Docker、docker compose、Go 二进制 + dist 静态资源 | 默认端口 6521 |
| 其他 | Go embed | 打包后端内置 assets |

## 项目边界

- 可基于公开功能描述重新实现上游 PRO 表中列出的面板能力，但不复制闭源代码、商业授权系统或付费运营后台。
- 不在初始化阶段大规模重构业务逻辑。
- 不删除上游 MIT 版权声明和许可证文本。
- 不承诺兼容上游旧配置、导入格式或闭源版本的数据结构。

## 项目约束

- 当前开源基线来自 Sun-Panel 开源版本，ZPanel 仓库已重新创建为独立 GitHub 仓库。
- 项目名、构建产物、Docker 服务名和文档应统一使用 `zpanel` / `ZPanel`。
- 优先保障 ZPanel 新版本可部署、可回滚；旧 Sun-Panel 数据迁移不作为初期目标。
- 任何涉及数据库结构、API 行为、认证权限的改动，都必须同步更新 project-log 对应文档。
- Docker 管理功能只面向管理员，并且必须明确提示部署环境需要 Docker CLI / Docker socket 权限。
- `project-log/` 是本地开发知识库，必须整体保持在 Git 忽略范围内，不随 GitHub 仓库发布。
