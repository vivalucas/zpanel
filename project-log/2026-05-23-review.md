# 2026-05-23 第三轮代码评审

## 评审范围

- 远端 `main` 已包含 `project-log/` 后，重新以云端版本为基线阅读关键文档。
- 重点复核首页初始化、系统监控、用户资料 / 密码修改、多账号快速切换、登录限流、后端初始化错误传播、前端 API 错误提示。
- 结合用户场景判断：ZPanel 是自托管 HomeLab / NAS 导航面板，既可能只在内网使用，也可能被用户反代到公网。

## 逐项复核结论

| 编号 | 类型 | 结论 | 优先级 | 复核依据 | 本轮处理 |
|------|------|------|--------|----------|----------|
| BUG-21 | CPU 监控卡片空值崩溃 | 真实 Bug | P1 | 模板中 `cpuState?.usages[0]` 只保护了 `cpuState`，当 `usages` 缺失或接口短暂返回空数据时仍会读取 `[0]` 崩溃。系统监控是首页常驻模块，用户无需特殊操作即可遇到。 | 修复 |
| BUG-22 | 数据库迁移 / 默认管理员初始化错误被忽略 | 真实 Bug | P1 | `DatabaseConnect()` 调用 `CreateDatabase()`、`NotFoundAndCreateUser()` 但原逻辑不检查错误。迁移失败、默认管理员创建失败时服务可能继续启动，留下不可登录或半初始化状态。 | 修复 |
| BUG-23 | 修改密码后前端仍保持登录态 | 真实 Bug | P1 | 后端修改密码后会撤销该用户所有 session，并删除当前 token 缓存；前端原逻辑只关闭弹窗和提示成功，下一次 API 调用才发现登录失效。 | 修复 |
| BUG-24 | 登录限流错误对前端不友好且记录不会清理 | 真实 Bug / 可用性问题 | P1 | 第二轮已修复并发竞态，但限流时直接返回 HTTP 429 的裸 JSON，不走统一 API 错误处理；同时过期 IP 记录没有清理，长时间公网暴露会累积无效记录。 | 修复 |
| BUG-25 | 首页初始化顺序存在竞态 | 真实 Bug | P2 | 首页挂载时用户信息刷新、列表加载、云端配置加载和标题设置并发执行。云端标题可能尚未加载就设置标题，导致用户配置的站点标题未及时生效。 | 修复 |
| BUG-26 | `updateLocalUserInfo()` 结果缺少防御 | 真实 Bug | P2 | `getAuthInfo` 失败或返回异常结构时，原函数可能抛出未处理异常或读取缺失的 `data.user`。该函数在首页和用户资料页都会调用。 | 修复 |
| BUG-27 | 系统监控编辑器把 Vue ref 写入业务数据 | 真实 Bug / 稳定性问题 | P2 | `currentMonitorData.value.extendParam = currentGenericProgressStyleExtendParam` 会把 ref 对象放入配置对象，依赖 Vue 序列化 / proxy 行为兜底，后续持久化和编辑回显脆弱。 | 修复 |
| BUG-28 | 拖拽列表 key 不稳定 | 真实 Bug / 交互问题 | P2 | 首页图标用数组 index 或可能重复的 `sort`，系统监控项用不存在的 `sort`。拖拽、排序、编辑时 Vue 复用 DOM 可能导致错位或状态串联。 | 修复 |
| BUG-29 | 昵称更新后多账号快速切换缓存不同步 | 真实用户体验问题 | P2 | 快速切换账号数据保存在 `ZPANEL_ACCOUNTS`，修改昵称只刷新当前 store，不更新本地账号缓存，用户再次打开快速切换仍看到旧昵称。 | 修复 |
| IMP-09 | 样式设置自动保存反馈过密 | 优化点成立 | P3 | 滑块 / 输入频繁变化会触发多次成功提示，和手动保存按钮语义重复。影响体验但不阻断功能。 | 延后 |
| IMP-10 | 自定义 CSS / JS 缺少恢复入口 | 优化点成立 | P3 | 自定义 CSS / JS 是明确功能，但错误脚本或样式可能让 UI 难以进入设置页恢复。需要设计安全模式或启动参数，改动范围较大。 | 延后 |
| IMP-11 | 搜索框同时承担网页搜索与导航过滤 | 优化点成立 | P3 | 用户心智上“搜索网页”和“过滤面板项目”是两个不同目标，当前混合入口可能误操作。属于产品交互优化。 | 延后 |

## 本轮修复策略

- 先处理 P1 和低风险 P2，避免继续累积线上可见故障。
- 不在本轮处理产品交互重设计、安全模式入口等较大范围优化，避免评审修复扩散为新功能重构。
- 修复后必须运行前端类型检查、lint、Go 测试，并对改动点再次做代码自检。

## 需要后续继续评估

- 自定义 CSS / JS 恢复路径应结合部署文档一起设计，避免用户把页面改坏后只能改数据库。
- 搜索框职责拆分需要先明确产品交互，不适合在 Bug 修复轮顺手改。
- 系统监控数据模型仍有 `any`，本轮只修真实引用问题，类型收敛应放到质量治理任务。
